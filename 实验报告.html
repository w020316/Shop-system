一、项目介绍

"微商城"是一个基于 Vue 3 框架开发的轻量级移动端电子商务应用，旨在为用户提供便捷的在线购物体验。项目采用 Vue 3 Composition API + TypeScript 进行开发，使用 Vue Router 4 进行路由管理，Pinia 进行全局状态管理，Vite 作为构建工具，Tailwind CSS 辅助样式开发。

项目实现了完整的电商购物流程，包括商品浏览与搜索、分类筛选、商品详情查看、购物车管理、用户注册与登录、订单创建与支付、收货地址管理、商品收藏、浏览历史记录、商品后台管理等功能。所有数据通过 localStorage 进行本地持久化存储，无需后端服务即可完整运行。

项目采用移动端优先的响应式设计，最大宽度480px居中展示，界面风格以暖橙色（#FF6B35）为主色调，搭配深灰文字和浅灰背景，营造活力与品质并存的购物氛围。

5.2 状态管理（Pinia Store）

5.2.1 商品Store（stores/product.ts）

商品Store管理商品列表、当前分类、搜索关键词等状态，通过computed属性实现分类筛选和搜索过滤，避免重复计算提升性能。吴宇威

import { defineStore } from 'pinia' import { ref, computed } from 'vue' import { products, categories, banners, type Product } from '@/data/products' export const useProductStore = defineStore('product', () => { // 响应式状态 const allProducts = ref<Product[]>(products) const currentCategory = ref('全部') const searchKeyword = ref('') const bannerList = ref(banners) // 计算属性：根据分类和关键词筛选商品 const filteredProducts = computed(() => { let result = allProducts.value if (currentCategory.value !== '全部') { result = result.filter(p => p.category === currentCategory.value) } if (searchKeyword.value.trim()) { const keyword = searchKeyword.value.trim().toLowerCase() result = result.filter(p => p.name.toLowerCase().includes(keyword) || p.description.toLowerCase().includes(keyword) || p.category.toLowerCase().includes(keyword) ) } return result }) // 计算属性：按销量排序的热门商品 const hotProducts = computed(() => [...allProducts.value].sort((a, b) => b.sales - a.sales).slice(0, 6) ) const categoryList = ref(categories) function setCategory(category: string) { currentCategory.value = category } function setSearchKeyword(keyword: string) { searchKeyword.value = keyword } function getProductById(id: number): Product | undefined { return allProducts.value.find(p => p.id === id) } return { allProducts, currentCategory, searchKeyword, filteredProducts, hotProducts, categoryList, bannerList, setCategory, setSearchKeyword, getProductById } })

5.2.2 购物车Store（stores/cart.ts）

购物车Store实现了完整的购物车逻辑，包括添加商品（同规格合并数量）、删除、数量更新、勾选/全选、合计金额计算，并通过localStorage实现数据持久化。吴宇威

export const useCartStore = defineStore('cart', () => { const items = ref<CartItem[]>([]) // 计算属性：已勾选商品 const checkedItems = computed(() => items.value.filter(item => item.checked)) // 计算属性：已勾选商品总价 const totalPrice = computed(() => checkedItems.value.reduce((sum, item) => sum + item.price * item.quantity, 0) ) // 计算属性：是否全选 const isAllChecked = computed(() => items.value.length > 0 && items.value.every(item => item.checked) ) // 加入购物车（同规格商品合并数量） function addToCart(item: Omit<CartItem, 'checked'>) { const existing = items.value.find( i => i.productId === item.productId && i.color === item.color && i.size === item.size ) if (existing) { existing.quantity += item.quantity // 同规格合并数量 } else { items.value.push({ ...item, checked: true }) } saveToStorage() } // 持久化到localStorage function saveToStorage() { localStorage.setItem('cart_items', JSON.stringify(items.value)) } // 从localStorage恢复数据 function loadFromStorage() { const data = localStorage.getItem('cart_items') if (data) { try { items.value = JSON.parse(data) } catch { items.value = [] } } } loadFromStorage() // 初始化时加载数据 // ... 其他方法 })

5.2.3 用户Store（stores/user.ts）

用户Store实现了注册、登录、退出、修改昵称等功能，注册用户列表和当前登录用户均通过localStorage持久化。登录时验证手机号格式和密码长度。甘润达

7.2 技术亮点

1. Composition API + TypeScript：全部组件采用Vue 3 Composition API和TypeScript开发，类型安全，代码可维护性高。

2. Pinia状态管理：7个独立Store模块化管理应用状态，computed属性实现派生状态计算，localStorage实现数据持久化。

3. 组件化设计：Toast通知、确认弹窗、图片兜底等公共组件复用性高，通过provide/inject实现跨组件通信。

4. 交互动画：fadeInUp入场动画、shake错误抖动、slideUp弹窗动画、轮播图淡入淡出等，提升用户体验。

5. 订单状态流转：pending→paid→shipped→completed完整生命周期，支付后自动模拟发货。

6. 响应式布局适配：移动端底部TabBar导航与PC端左侧固定侧边栏自动切换，通过监听窗口resize事件实现响应式布局。

7. 路由鉴权守卫：通过router.beforeEach全局前置守卫和meta.requiresAuth标记，实现未登录用户自动重定向到登录页，登录后跳回原页面。

8. 数据一致性保障：商品详情页切换商品时自动重置选择状态，订单确认页组件卸载时清理定时器防止内存泄漏，全局统一使用Toast替代原生alert。

8.2 个人总结

吴宇威

本次大作业中，我主要负责项目整体架构设计和核心模块开发。通过从零搭建Vue 3 + TypeScript + Pinia + Vue Router技术栈，深入理解了Composition API的响应式原理和状态管理的最佳实践。在开发购物车模块时，同规格商品合并数量的逻辑设计让我对响应式数据的更新机制有了更深的认识。全局Toast通知采用provide/inject模式，避免了props逐层传递的繁琐。项目开发过程中最大的收获是理解了前端框架的组件化思想——将复杂页面拆分为独立、可复用的组件，通过清晰的接口通信，使代码结构清晰、易于维护。

甘润达

我负责用户认证和订单系统模块的开发。在实现注册登录功能时，我学习了正则表达式验证手机号格式的方法，以及如何在Pinia Store中管理用户状态和持久化数据。订单系统的状态流转设计（pending→paid→shipped→completed）让我理解了状态机的设计思想，通过setTimeout模拟发货流程，使应用无需后端也能展示完整的购物流程。表单验证的抖动动画效果增强了用户交互体验，让我认识到细节设计对用户体验的重要性。

姚泽楷

我负责分类浏览页和地址管理模块的开发。分类页的左右双栏布局是一个经典的电商UI模式，左侧固定宽度导航栏配合右侧滚动内容区，通过CSS的flex布局实现，选中项的橙色指示条使用::before伪元素定位。地址管理模块涉及CRUD完整操作，设为默认地址时需要取消其他地址的默认状态，这个逻辑让我理解了状态一致性的重要性。收藏功能通过Pinia Store统一管理，在商品详情页和个人中心页都能访问和操作，体现了集中式状态管理的优势。

黄显杰

我负责商品管理、浏览历史、404页面和测试工作。商品管理页实现了作业要求的扩展功能——添加和删除商品，添加时自动生成占位图片URL，删除时二次确认防止误操作。浏览历史功能通过localStorage记录用户浏览过的商品ID，最多保留20条，去重处理确保同一商品不会重复记录。在编写24个测试用例的过程中，我系统地验证了所有功能模块，确保每个交互流程都能正常工作，这让我认识到测试在软件开发中的重要性。

8.3 收获与体会

通过本次大作业，我们小组深入实践了Vue 3框架的核心技术，包括Composition API、响应式系统、组件化开发、路由管理和状态管理。项目从需求分析、功能设计、代码实现到测试验证的完整开发流程，让我们体验了真实的前端项目开发过程。团队协作中，我们通过合理的分工和有效的沟通，确保了项目按时高质量完成。这次实践不仅巩固了课堂所学知识，更培养了我们的工程化思维和团队合作能力。
